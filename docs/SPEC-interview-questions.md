# AI 면접 질문 생성기 SPEC

작성일: 2026-03-29
작성: 제품기획팀 (플랫폼총괄 산하)
상태: 초안 (Planner)

---

## 개요

채용공고와 이력서를 입력하면, AI가 "이 면접관이라면 이 지원자에게 어떤 질문을 던질까"를 시뮬레이션하는 도구다.

단순한 질문 목록 생성이 아니다. 질문마다 **면접관의 의도**, **이상적인 답변 방향**, **이 질문이 나올 가능성(빈도 등급)**을 함께 제공한다. 지원자가 "왜 이 질문이 나오는가"를 이해하고 준비할 수 있도록 설계한다.

### 포지셔닝
- 바로도구 내 "취업/직장인" 카테고리 핵심 도구
- AI 핵심역량 추출기와 쌍으로 쓰이는 준비 도구 세트
- 무료, 회원가입 없음, Groq 무료 티어(LLaMA 3.1) 활용

### 타깃 사용자
- 서류 합격 후 면접을 앞둔 취업 준비생
- 이직 면접을 준비하는 직장인 (경력직 포함)
- AI 핵심역량 추출기로 역량을 정리한 직후, 다음 단계가 필요한 사용자

---

## 디자인 방향

### 원칙: "면접관 시뮬레이터" 프레임

도구가 아니라 **역할극(Role-play) 인터페이스**처럼 느껴져야 한다. 질문 목록을 나열하는 표 형태 대신, 면접관이 실제로 질문을 던지는 카드 UI로 구성한다.

### 톤 앤 무드

- 긴장감보다 "내가 준비됐다"는 자신감을 주는 톤
- 질문 카드: 미묘한 그림자, 약간의 depth — 실제 카드를 집는 느낌
- 결과 화면은 인쇄하고 싶어지는 구성 (포트폴리오 느낌)
- 다크모드에서도 무거운 배경 대신 차분한 네이비 계열 유지

### AI Slop 금지 원칙

다음은 절대 사용하지 않는다.

- "AI가 분석했습니다" 같은 기계적 문구
- 별점 UI, 원형 게이지, 퍼센트 숫자 남용
- 동일한 폰트 크기의 나열형 리스트

대신 사용하는 것.

- 면접관 페르소나 이름 (ex. "마케팅팀 팀장 시뮬레이션")
- 질문 카드에 빈도 등급을 색상 태그로 표시 (빨강/주황/파랑)
- 답변 힌트를 접혔다 펼쳐지는 아코디언으로 처리

### 레이아웃 흐름

```
[입력 패널]
  채용공고 | 이력서 (탭 전환)
  생성 버튼

[결과 패널]
  요약 배너 → 면접 카드 목록 → CTA (핵심역량 추출기)
```

입력과 결과가 같은 페이지에서 세로 흐름으로 이어진다. 새 페이지로 이동하지 않는다.

## 기능 목록

---

### 기능 1: 채용공고 구조화 입력

**설명**

채용공고를 하나의 텍스트 덩어리로 붙여 넣는 것이 아니라, 담당업무 / 자격요건 / 우대사항 / 조직문화 네 개 필드로 나눠 입력받는다.

기존 AI 핵심역량 추출기가 같은 방식을 사용 중이므로 UX가 익숙하다. 두 도구를 연결할 때 채용공고 정보를 재사용하기 쉬운 구조이기도 하다.

**사용자 스토리**

As a 지원자, I want 채용공고를 항목별로 넣을 수 있도록 so that AI가 어떤 부분에서 질문이 나왔는지 연결해서 볼 수 있다.

**포함 요소**

- 담당업무, 자격요건, 우대사항, 조직문화 — 각각 독립된 텍스트 에리어
- 각 필드 상단에 짧은 플레이스홀더 (예: "직무기술서에서 '주요업무' 섹션을 붙여넣으세요")
- 필드 중 하나만 입력해도 작동 (나머지는 선택)
- 이미지 OCR 지원: 채용공고 스크린샷을 올리면 필드 자동 채우기 (기존 핵심역량 추출기 OCR 로직 재사용)

---

### 기능 2: 이력서 입력 (텍스트 + 이미지)

**설명**

이력서 텍스트를 직접 붙여 넣거나, 이미지/PDF 스크린샷을 OCR로 처리한다.

**사용자 스토리**

As a 지원자, I want 이력서를 텍스트나 이미지 어느 방식으로든 입력할 수 있도록 so that 도구 진입 장벽을 낮출 수 있다.

**포함 요소**

- 탭 UI: "직접 입력" / "이미지 붙여넣기"
- 직접 입력: 큰 텍스트 에리어 (최소 10줄 높이)
- 이미지 입력: 드래그앤드롭 또는 클릭 업로드, OCR로 텍스트 자동 추출
- 업로드 중 로딩 상태 표시 ("이력서를 읽는 중...")

---

### 기능 3: AI 면접 질문 생성 (핵심 기능)

**설명**

채용공고와 이력서를 교차 분석하여 면접관이 실제로 할 만한 질문을 생성한다. 단순 키워드 매칭이 아니라 "이 이력서에서 이 채용공고의 요건을 봤을 때 면접관이 어떤 불확실성을 느끼는가"를 추론하는 방식이다.

**사용자 스토리**

As a 지원자, I want 내 이력서와 이 채용공고를 분석한 예상 질문을 받고 싶다 so that 막연하게 준비하지 않고 핵심을 집중 연습할 수 있다.

**포함 요소**

- 생성 버튼 클릭 시 단계별 로딩 메시지 표시
  - "채용공고를 읽는 중..."
  - "이력서와 교차 분석 중..."
  - "면접 질문을 구성 중..."
- 질문 15~20개 생성 (카테고리별로 구분)
- Groq API (LLaMA 3.1) 호출, 스트리밍 응답 지원

---

### 기능 4: 질문 카테고리 분류

**설명**

생성된 질문을 카테고리별로 묶어 탭으로 분리한다. 탭 전환만으로 "직무 역량 집중 연습", "인성 집중 연습" 등 모드 전환이 가능하다.

**사용자 스토리**

As a 지원자, I want 질문을 종류별로 분리해서 보고 싶다 so that 남은 준비 시간에 따라 어떤 질문 유형을 먼저 연습할지 선택할 수 있다.

**포함 요소**

- 카테고리: 직무 역량 / 경험 검증 / 지원 동기 & 조직문화 / 돌발 & 압박
- 탭 상단에 각 카테고리별 질문 수 뱃지 표시
- "전체 보기" 탭도 기본 제공
- 카테고리별로 색상 구분 (직무=파랑, 경험=초록, 지원동기=주황, 돌발=빨강)

---

### 기능 5: 질문 카드 — 빈도 등급 + 면접관 의도

**설명**

질문 하나하나가 단순 텍스트가 아니라 구조화된 카드로 표시된다. 카드에는 질문 본문 외에 "이 질문이 나올 가능성(빈도 등급)"과 "면접관이 이 질문으로 무엇을 보려는지"가 포함된다.

**사용자 스토리**

As a 지원자, I want 각 질문이 왜 나오는지 알고 싶다 so that 답변을 외우는 것이 아니라 면접관의 관점으로 이해하고 대비할 수 있다.

**포함 요소**

- 질문 카드 구성:
  - 상단 좌측: 카테고리 컬러 태그
  - 상단 우측: 빈도 등급 뱃지 (높음 / 보통 / 낮음)
  - 본문: 질문 텍스트 (굵게, 크게)
  - 하단: 면접관 의도 한 줄 요약 (회색 작은 글씨)
- 빈도 등급 색상: 높음=빨강, 보통=주황, 낮음=파랑
- 카드 클릭 시 "답변 힌트" 아코디언 펼침

---

### 기능 6: 답변 힌트 아코디언

**설명**

각 질문 카드 하단의 "답변 힌트 보기" 버튼을 클릭하면 AI가 생성한 답변 방향이 펼쳐진다. 답을 그대로 주는 것이 아니라 "무엇을 언급해야 효과적인가"를 가이드한다.

**사용자 스토리**

As a 지원자, I want 어떻게 답하면 좋은지 방향을 알고 싶다 so that 모범 답안을 외우는 것이 아니라 나만의 언어로 답변을 구성할 수 있다.

**포함 요소**

- 아코디언 기본 상태: 접혀있음 (결과 페이지 스캔이 쉽도록)
- 힌트 내용:
  - 핵심 키워드 2~3개 (답변에 반드시 포함할 것)
  - 이 지원자의 이력서에서 활용하면 좋은 경험 연결
  - 피해야 할 답변 패턴 1가지
- 힌트 텍스트는 불릿이 아닌 자연스러운 문장체

---

### 기능 7: 결과 요약 배너

**설명**

질문 카드 목록 상단에 "이번 면접의 핵심 포인트"를 한눈에 보여주는 배너를 배치한다.

**사용자 스토리**

As a 지원자, I want 전체 질문을 읽기 전에 핵심만 먼저 파악하고 싶다 so that 제한된 시간에 우선순위를 잡을 수 있다.

**포함 요소**

- 배너 구성:
  - 생성된 총 질문 수
  - 가장 많이 출현한 카테고리 1~2개 강조 (예: "직무 역량 질문이 집중됩니다")
  - 이력서에서 AI가 발견한 "면접관이 파고들 가능성 있는 공백" 1~2개 한 줄 알림
- 배너 디자인: 연한 배경에 아이콘 없이 텍스트 중심, 과도한 시각 요소 배제

---

### 기능 8: 핵심역량 추출기 크로스셀 CTA

**설명**

결과 하단에 AI 핵심역량 추출기로 연결하는 CTA 블록을 배치한다. "면접 질문 준비 완료 → 다음 단계: 면접 답변에 쓸 핵심역량 정리" 흐름으로 자연스럽게 이어진다.

**사용자 스토리**

As a 지원자, I want 면접 질문 준비 후 자연스럽게 다음 준비 단계로 이동하고 싶다 so that 바로도구 안에서 준비를 이어갈 수 있다.

**포함 요소**

- 위치: 질문 목록 최하단 (광고 슬롯 위)
- CTA 문구: "답변에 쓸 핵심역량, 아직 정리 안 하셨나요?" (뻔한 "관련 도구" 헤더 금지)
- AI 핵심역량 추출기 카드 1개 (도구 이름, 설명 한 줄, 이동 버튼)
- 역방향 CTA도 설계: 핵심역량 추출기 결과 페이지 하단에도 면접 질문 생성기 링크 추가

---

### 기능 9: 결과 인쇄 / 복사

**설명**

생성된 질문 전체를 클립보드에 복사하거나, 인쇄 최적화 레이아웃으로 출력할 수 있다.

**사용자 스토리**

As a 지원자, I want 질문 목록을 따로 저장하고 싶다 so that 면접 전날 오프라인에서도 복습할 수 있다.

**포함 요소**

- "전체 복사" 버튼: 질문 + 답변 힌트를 마크다운 형태로 클립보드에 복사
- "인쇄하기" 버튼: `@media print` CSS로 광고, 헤더, CTA 숨기고 질문 카드만 출력
- 복사 완료 시 토스트 알림 ("클립보드에 복사되었습니다")

---

### 기능 10: 재생성 및 질문 수 조절

**설명**

생성된 결과가 마음에 들지 않거나 더 많은 질문이 필요할 때 조절할 수 있다.

**사용자 스토리**

As a 지원자, I want 질문 수와 난이도를 조절해서 다시 생성하고 싶다 so that 면접까지 남은 시간에 맞는 분량으로 준비할 수 있다.

**포함 요소**

- 결과 상단에 설정 바: 질문 수 (10 / 15 / 20개), 난이도 (기본 / 심화)
- "재생성" 버튼: 같은 입력으로 다른 각도의 질문 생성
- 심화 모드: 돌발 질문, 상황 기반 시나리오 질문 비율 높임

## 연결 구조 (기존 도구와의 관계)

```
[AI 핵심역량 추출기]
  결과 하단 CTA ---> [AI 면접 질문 생성기]

[AI 면접 질문 생성기]
  결과 하단 CTA ---> [AI 핵심역량 추출기]
```

두 도구는 같은 채용공고 + 이력서 입력 구조를 공유한다. 향후 "채용공고 저장" 기능이 생기면 입력을 공유할 수 있는 기반이 된다.

## 비용 / 인프라 메모

- Groq 무료 티어 (LLaMA 3.1 70B): 분당 14,400 토큰
- 요청당 예상 입력 토큰: 약 800~1,500 (채용공고 + 이력서)
- 요청당 예상 출력 토큰: 약 1,200~2,000 (질문 15개 + 힌트)
- 일 1,000회 요청 기준 → 무료 티어 내 충분히 수용 가능
- 추가 비용 없음 (기존 Vercel + Groq 인프라 그대로 사용)
