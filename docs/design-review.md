## 검수 전 현황 확인

현재 사이트 코드를 직접 확인한 결과, 디자인팀의 "AI slop 패턴" 진단은 **전부 사실**이다.

- HomeClient.tsx: 중앙 정렬 히어로 + 체크마크 3개 나열 (그대로)
- ToolCard.tsx: `rounded-xl + shadow-sm + hover:-translate-y-0.5` (Tailwind 기본 카드 패턴 그대로)
- CategoryFilter.tsx: `rounded-full` 알약 버튼 수평 나열 (진부한 패턴 그대로)
- 색상: `text-primary`가 Tailwind blue-600 계열 (확인 필요하나 높은 확률)
- 도구 15개 운영 중, 카테고리 6개

현재 상태가 개선이 필요하다는 판단에는 동의한다. 이하 각 안에 대한 검수 결과.

## 안 1: "조판실" (Typesetting Room) 검수

### 장점
1. **취업 도메인과의 정합성이 가장 높다.** 공문서, 이력서, 서류를 다루는 사이트에 명조체 + 크림 배경은 의미적으로 연결된다. 사용자가 "여기가 진짜 취업 도구 사이트구나"라고 느끼게 만드는 힘이 있다.
2. **색상 차별화가 확실하다.** #C8401A(교정 빨간 펜) + #F5F2EC(크림)은 국내 유틸리티 사이트에서 볼 수 없는 조합이다. SaaS 파란색에서 완전히 벗어난다.
3. **다크모드 전환이 자연스럽다.** 크림(#F5F2EC) -> 잉크 블랙(#1A1814) 전환은 밝은 종이 -> 어두운 인쇄소라는 일관된 세계관을 유지한다.
4. **구현 난이도가 가장 낮다.** 격자선 노출은 CSS background-image linear-gradient로 가능하고, 비대칭 히어로는 flex/grid로 충분하다. 복잡한 애니메이션이 없다.
5. **Nanum Myeongjo + IBM Plex Sans KR**: 둘 다 Google Fonts에서 무료 제공, 한글 완벽 지원.

### 단점/리스크
1. **명조체 헤드라인은 20대 초반에게 "올드"하게 느껴질 수 있다.** 명조체 = 신문 = 구세대라는 연상이 존재한다.
2. **격자선 노출이 지저분하게 보일 위험.** opacity와 색상을 정밀하게 조정하지 않으면 모눈종이처럼 느껴진다.
3. **테두리선만 있는 카드(outline only)는 모바일에서 터치 영역 인지가 약할 수 있다.** 배경색이 없으면 카드 경계가 불분명해질 위험.
4. **HOT/NEW 스탬프 형태가 과하면 키치해진다.** 적절한 수준 유지가 디자인 실행에 달려 있다.

### 유저 반응 예측
- 취준생: "다른 사이트랑 확실히 달라서 기억에 남는다"
- 직장인: "깔끔하고 신뢰감 있다"
- 20대 초반 일부: "약간 딱딱하다"

### 점수: 7.5 / 10

## 안 2: "터미널 도구" (Terminal Tools) 검수

### 장점
1. **커맨드팔레트(Cmd+K) 검색은 킬러 기능이 될 수 있다.** 국내 어떤 유틸리티 사이트도 이걸 제공하지 않는다. 파워유저에게 "이 사이트 진짜 잘 만들었네"라는 인상을 준다.
2. **다크모드 완성도가 가장 높다.** 다크 퍼스트 설계이므로 다크모드가 디폴트처럼 작동한다. 2026년 대다수 사용자가 다크모드를 쓰는 환경에서 유리하다.
3. **Space Grotesk + JetBrains Mono 조합은 시각적 개성이 매우 강하다.** 국내 사이트에서 거의 사용되지 않는 폰트 조합이라 차별화 효과가 크다.
4. **히어로의 가짜 터미널 타이핑 애니메이션은 눈길을 확실히 끈다.** 처음 방문 시 3초 이내에 "여기 뭔가 다르다"는 인상을 준다.
5. **해시태그 필터(#취업 #AI #금융)는 SNS 네이티브 세대에게 직관적이다.**

### 단점/리스크
1. **타겟 미스매치가 가장 큰 리스크다.** 바로도구의 핵심 타겟은 "취준생/직장인 전체"인데, 터미널 감성은 "개발직군 취준생"에게만 친숙하다. 비개발 직군 취준생이 "이거 개발자용 사이트인가?"라고 오해할 수 있다.
2. **터미널 타이핑 애니메이션 구현 복잡도가 높다.** 타이핑 → 결과 출력 → 루프 애니메이션 + 모바일 대응까지 하면 개발 시간이 상당히 소요된다.
3. **커맨드팔레트가 모바일에서는 의미가 약하다.** Cmd+K는 데스크톱 전용이고, 모바일에서는 일반 검색과 차이가 없다. 트래픽의 70%+ 가 모바일일 가능성이 높은 타겟층에서 핵심 차별점이 사라진다.
4. **2열 리스트뷰는 도구 15개에서는 적합하지만, 30개 이상이면 스크롤이 과도하게 길어진다.**
5. **`baro://` 로고 형식은 비개발자에게 의미가 전달되지 않는다.**

### 유저 반응 예측
- 개발직 취준생: "대박, 이런 취업 사이트 처음이야" (강한 호감)
- 비개발직 취준생: "이거 개발자용인가?" (이탈 위험)
- 직장인: "세련되긴 한데 나한테 맞는 건지 모르겠다"

### 점수: 7.0 / 10

## 안 3: "도구 잡지" (Tool Magazine) 검수

### 장점
1. **에디토리얼 그리드(Feature Card + Regular Card 혼합)는 시각적 흥미가 가장 높다.** 한 화면에서 대형 카드가 눈을 끌고, 작은 카드는 탐색을 유도한다. "발견의 즐거움"이라는 UX 목표가 구조적으로 뒷받침된다.
2. **형광 노랑(#FFF176) + 잡지 레드(#FF3B3B) 조합은 강렬하다.** 스크린샷 하나만으로도 다른 사이트와 구별된다.
3. **Bebas Neue 대문자 타이틀은 시선을 강하게 잡는다.** "BARO DOGU" 대문자 히어로는 로고 없이도 브랜드 인지를 만든다.
4. **도구가 늘어날수록 레이아웃이 더 풍성해지는 구조.** 장기적으로 도구 50개+ 시 가장 빛나는 안이다.
5. **형광 하이라이트 마커 효과는 구현이 단순하다.** CSS background linear-gradient로 밑줄 형태의 형광 마커를 1줄로 구현 가능하다.

### 단점/리스크
1. **불규칙 그리드 구현이 복잡하다.** Feature Card와 Regular Card의 반응형 배치를 모바일까지 깔끔하게 처리하려면 CSS Grid 고급 기법이 필요하다. 모바일에서 대형 카드/소형 카드 구분이 사라지면 차별점이 반감된다.
2. **도구 15개로는 잡지 느낌이 나지 않는다.** 에디토리얼 그리드는 콘텐츠가 풍부해야 의미가 있는데, 현재 규모에서는 빈 잡지처럼 보일 수 있다.
3. **형광 노랑 남용 시 가독성이 떨어진다.** 강조가 많으면 강조가 아니게 된다. 가이드라인 없이 적용하면 산만해질 위험.
4. **"편집장 추천" 같은 큐레이션 섹션은 수동 관리가 필요하다.** 자동화가 아닌 콘텐츠 편집이 지속적으로 필요한 구조라 1인 기업에 부담.
5. **Bebas Neue는 영문 전용 폰트.** 한글 헤드라인에는 Noto Serif KR을 쓰게 되는데, 두 폰트의 성격 차이로 영문/한글 혼합 시 시각적 통일감이 약해질 수 있다.

### 유저 반응 예측
- 취준생: "디자인 예쁘다, 뭔가 잡지 보는 것 같다"
- 직장인: "깔끔하고 흥미롭다"
- 재방문: 콘텐츠 부족하면 "처음엔 좋았는데 볼 게 없다"

### 점수: 6.5 / 10

## 최종 결론

### 추천 안: 안 1 "조판실" (하이브리드 보강)

### 선택 이유

1. **타겟 정합성**: 바로도구의 핵심 타겟은 "취준생 + 직장인 전체"이다. 안 2는 개발직에 편향되고, 안 3은 콘텐츠 부족 시 빈 잡지가 된다. 안 1은 "취업 서류를 다루는 전문 작업실"이라는 컨셉이 타겟 전원에게 통한다.

2. **구현 현실성**: 1인 기업 체제에서 복잡한 애니메이션이나 불규칙 그리드보다, CSS만으로 완성도를 높일 수 있는 안이 유리하다. 안 1은 기존 컴포넌트(ToolCard, CategoryFilter, ToolGrid)의 스타일만 변경하는 수준으로 전환 가능하다.

3. **모바일 우선**: 20~30대 취준생 트래픽의 대부분이 모바일이다. 안 1의 비대칭 히어로 + 밑줄 카테고리 탭은 모바일에서도 자연스럽게 작동한다. 안 2의 커맨드팔레트와 안 3의 불규칙 그리드는 모바일에서 핵심 차별점을 잃는다.

4. **브랜드 차별화**: 명조체 헤드라인 + 크림 배경 + 교정 빨간 펜 색상은 국내 유틸리티/취업 사이트 어디에도 없다. "바로도구 = 크림색 작업실"이라는 시각 기억이 남는다.

### 안 2에서 차용할 요소 (하이브리드)

안 2의 핵심 장점 중 구현 가능한 것을 안 1에 흡수한다:

- **커맨드팔레트 검색 (Cmd+K)**: 안 1의 풀와이드 검색 라인 대신, 커맨드팔레트를 추가 UX로 제공한다. 구현 자체는 cmdk 라이브러리(npmjs.com/package/cmdk)로 비교적 간단하다. 모바일에서는 기존 검색 UI를 유지하고, 데스크톱에서만 Cmd+K를 활성화하면 된다.
- **다크모드 우선 고려**: 안 1의 다크모드(잉크 블랙 #1A1814)를 기본 경험에 가깝게 다듬는다. 안 2의 다크 퍼스트 철학을 일부 차용하여, 다크모드 시 더 몰입감 있는 작업실 느낌을 낸다.

### 적용 시 주의사항

1. **명조체 사용 범위를 제한한다.** 헤드라인(h1, h2)에만 Nanum Myeongjo를 적용하고, 본문/버튼/라벨은 반드시 IBM Plex Sans KR로 유지한다. 명조체가 과다하면 "관공서 사이트" 느낌이 된다.

2. **격자선 노출은 히어로 영역에만 한정한다.** 페이지 전체에 격자선을 넣으면 산만하다. 히어로 배경에만 연한 격자선을 깔고, 도구 리스트 영역부터는 깨끗한 배경으로 전환한다.

3. **테두리선 카드(outline only)에 호버 시 배경색을 추가한다.** 기본은 outline only로 차별화하되, 호버 시 크림색 배경이 살짝 들어오게 하여 인터랙션 피드백을 확보한다. 모바일 터치 시에도 active 상태에서 배경색을 보여준다.

4. **HOT/NEW 스탬프의 회전 각도를 -3deg ~ 3deg 이내로 제한한다.** 과도한 회전은 키치하다. 미세한 기울기 + opacity 0.85 정도로 절제한다.

5. **폰트 로딩 최적화가 필수.** Nanum Myeongjo + IBM Plex Sans KR + IBM Plex Mono 세 가지를 Google Fonts에서 불러와야 한다. next/font를 활용하여 font-display: swap으로 설정하고, 필요한 weight만 로드한다 (Myeongjo: 400, 700 / Plex Sans: 400, 500, 600 / Plex Mono: 400).

6. **기존 14개 도구 페이지의 스타일 통일 작업이 동반되어야 한다.** 메인 페이지만 리뉴얼하고 도구 페이지는 그대로 두면 경험이 단절된다. 도구 페이지의 헤더/색상/폰트를 동시에 전환해야 한다.

7. **Primary 색상(#C8401A) 접근성 확인 필요.** 크림 배경(#F5F2EC) 위에서 WCAG AA 기준 대비율을 충족하는지 검증한다. 필요 시 약간 어둡게 조정한다.

### 디자인팀에 추가 요청 사항

- 안 1 기반 하이브리드의 HTML/CSS 프로토타입 제작 요청
- 커맨드팔레트(Cmd+K) 모달 디자인을 안 1 색상 체계에 맞춰 추가 시안 제작
- 도구 상세 페이지(예: 연봉 계산기) 1종에 대한 리뉴얼 시안도 함께 제출 요청 (메인만 바꾸고 상세가 다르면 안 됨)
- 모바일 와이어프레임 별도 제출 요청
